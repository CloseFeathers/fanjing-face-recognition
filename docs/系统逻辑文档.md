## 1. 系统总览

本系统是一个**实时人脸识别与身份管理流水线**，从摄像头或视频文件中读取画面，逐帧执行处理。

**双线程架构：**

```
主线程: 读帧 → 检测 → 跟踪 → 画框 → JPEG推流（保证FPS流畅）
异步线程1 (IdentityWorker): 对齐 → 质量 → 采样 → embedding → 匹配 → 身份判定
异步线程2 (MouthWorker): MediaPipe → BiSeNet遮挡 → XGBoost说话检测（可选）
```

主线程**不等待**任何身份/说话结果，只读取最新的缓存标签来显示。这确保了新人入画或重新识别时 FPS 不会波动。

**核心设计原则：**
- **Fail-open（失败不阻塞）**：任何下游模块出错都不影响视频流
- **逐层递进**：每个模块只在上一个模块产出有效结果时才触发
- **异步解耦**：身份计算和画面渲染完全分离

## 2. Module 0：视频输入层

**职责**：统一摄像头和视频文件两种输入，对外提供标准的「帧」对象。

### 2.1 帧对象（Frame）

每一帧包含：
- **图像数据**：BGR 格式的像素矩阵
- **时间戳**：毫秒级。摄像头使用系统单调时钟，视频使用容器自带的播放位置
- **帧序号**：从 0 递增
- **来源标识**：如 `camera:0` 或 `file:demo.mp4`
- **宽高**、**丢帧数**

### 2.2 摄像头源

采用**生产者-消费者模型**，帧缓冲区大小为 1，只保留最新帧。主线程总是拿到最新画面，不会延迟堆积。

### 2.3 视频文件源

按帧顺序逐帧读取，时间戳取自视频容器。提供 `realtime` 开关控制是否限速播放。

## 3. Module 1：人脸检测

**职责**：在每帧画面中找到所有人脸的位置、置信度和关键点。

### 3.1 检测器：SCRFD

使用 **SCRFD** 模型（`det_10g.onnx`），通过 ONNX Runtime 推理。

- **输入**：原图按比例缩放到指定尺寸（如 640×640），letterbox 填充
- **输出**：每张人脸的 bbox [x1,y1,x2,y2]、score (0~1)、5个关键点（左眼、右眼、鼻尖、左嘴角、右嘴角）
- **线程限制**：`intra_op_num_threads = max(cpu_count - 2, 4)`，为异步线程的 ArcFace 推理留出 CPU 核心，消除冷启动 FPS 波动

### 3.2 四阈值分离架构

| 阈值 | 名称 | 默认值 | 作用 |
|------|------|--------|------|
| **A** | `detector_emit_min_det` | 0.15 | 检测器输出到 tracker 的最低分数 |
| **B** | `track_new_high_thres` | 0.50 | 只有高分框可以创建新轨迹 |
| **C** | `track_update_low_thres` | 0.15 | 低分框可以更新已有/丢失轨迹 |
| **D** | `sample_min_det` | 0.60 | 进入对齐/嵌入/身份识别的最低分数 |

弱检测框（0.15~0.50）被 tracker 用来维持跟踪连续性，但不能创建新轨迹、也不能进入嵌入流程。

## 4. Module 2：多目标跟踪

**职责**：将逐帧检测关联成连续轨迹，分配稳定的 `track_id`。

### 4.1 算法：BoT-SORT（四阶段匹配）

1. **预测**：Kalman 滤波器预测所有轨迹位置
2. **第一阶段**：高分检测 vs 活跃轨迹（IoU 匹配）
3. **第二阶段**：低分弱框 vs 未匹配的活跃轨迹（弱框维持已有轨迹）
4. **第三阶段**：未匹配高分检测 vs 丢失轨迹（恢复）
5. **第四阶段**：低分弱框 vs 丢失轨迹（弱框续命）
6. **新建**：仍未匹配的高分检测（score ≥ B）创建新轨迹。低分框永远不能创建新轨迹

### 4.2 轨迹生命周期

```
新检测 → TENTATIVE（试探） → CONFIRMED（确认） → LOST（丢失） → REMOVED（移除）
```

- TENTATIVE → CONFIRMED：连续 `min_hits`（默认 3）帧被匹配
- CONFIRMED → LOST：检测帧中未匹配
- LOST → CONFIRMED：重新被匹配（复活）
- LOST → REMOVED：超过 `max_age`（默认 30）个检测帧未匹配

### 4.3 非检测帧处理

当 `det_every_n > 1` 时，非检测帧仅执行 Kalman 预测推算位置，大幅提升 FPS。

## 5. Module 3：人脸对齐 + 质量评估 + 采样

**职责**：标准化人脸、评估质量、保留高质量样本。在 **IdentityWorker 异步线程**中执行。

### 5.1 人脸对齐

使用 5 个关键点 + Umeyama 算法，将任意角度的人脸映射到标准 **112×112** 正脸图。支持旋转/倾斜的人脸（如躺着使用）。

### 5.2 质量评估（Quality Gate）

| 维度 | 说明 | 默认阈值 |
|------|------|----------|
| **检测置信度** | SCRFD 原始 score | ≥ 0.60 |
| **人脸面积** | 原始 bbox 像素面积 | ≥ 900 px²（30×30） |
| **清晰度** | 对齐后图的 Laplacian 方差 | ≥ 15.0 |
| **关键点合理性** | 双眼距离 ≥ 20px | 通过检查 |

四维全部通过才合格。同时计算加权综合分数用于排序。

### 5.3 采样管理

每条轨迹维护 best-K 样本缓存（默认最多 10 张），新样本质量更高则替换最差的。

## 6. Module 4：人脸嵌入 + 模板聚合 + 人员匹配

**职责**：人脸图转向量，聚合模板，会话内身份匹配。在 **IdentityWorker 异步线程**中执行。

### 6.1 嵌入提取（ArcFace）

- 模型：`w600k_r50.onnx`，输出 512 维归一化向量
- 线程限制：`intra_op_num_threads=2`，不与 SCRFD 争抢
- 两张脸的余弦相似度：同一人通常 > 0.5，不同人通常 < 0.3

### 6.2 模板聚合

收集同一轨迹的多次嵌入，用质量加权平均聚合。至少 3 个嵌入后才生成模板。

### 6.3 人员匹配（Person Registry）

新 track 模板与已知 person 模板计算余弦相似度：
- top1 ≥ 0.4 且 margin ≥ 0.1 → 匹配成功，绑定已有 person
- 否则 → 创建新 person

匹配后 person 模板滑动平均更新。

## 7. Module 5：三态身份判定 + 自动注册

**职责**：判定每个 person 是已知身份还是陌生人，自动注册和跨 session 持久化。

### 7.1 三态身份判定

对每个 session person，与 **RegisteredPersonDB（已注册身份库）** 比对：

| 状态 | 缩写 | 触发条件 |
|------|------|----------|
| **KNOWN_STRONG** | KS | top1 相似度 ≥ 0.55 且 margin ≥ 0.10 |
| **AMBIGUOUS** | AMB | 相似度在 [0.35, 0.55) 或 margin 不足 |
| **UNKNOWN_STRONG** | US | 相似度 < 0.35，或已注册库为空 |

### 7.2 自动注册

- **UNKNOWN_STRONG 且 register_ready** → 自动注册为 R#y（新的已注册身份），后续 session 中被识别为 KS
- **register_ready 综合评分**：样本数、一致性、平均质量、持续时间的加权分数 ≥ 阈值
- **AMBIGUOUS 超时注册**：停留在 AMB 超过 15 秒且 register_ready → 强制注册

### 7.3 KNOWN_STRONG 模板更新

KS 的 person 会保守更新其已注册身份的模板（权重 0.2），使已知库随使用越来越精确。

### 7.4 自动合并

模板更新后触发 `check_and_merge()`：如果两个已注册身份相似度 ≥ 0.75，将较新的合并到较旧的，更新所有映射。

### 7.5 持久化

- `RegisteredPersonDB` 在 pipeline 停止时自动 `save()`，启动时自动 `load()`
- 存储路径：`data/registered_db/`

## 8. Credit Gate：信用积分门控

**职责**：防止偶然通过 Quality Gate 的帧直接进入 embedding。不删除任何轨迹。

每条轨迹维护 `face_valid_credit`（浮点，初始 0）：

- Quality 通过 → credit += 1.0
- Quality 未通过 → credit -= 0.5
- 下限 0，上限 10.0
- credit ≥ 3.0 时才允许 embedding

典型场景：3 帧连续好 → 信用 3.0 达标；远处走近 → 信用自然积累 → 自动达标。

## 9. 说话检测模块

**职责**：判断每个 track 的人是否在说话。**可选功能**，前端有开关，默认关闭。

### 9.1 架构

在独立的 **MouthWorker 异步线程** 中运行，不影响主线程 FPS：

```
face_crop → MediaPipe (blendshapes + yaw) → BiSeNet (遮挡) → XGBoost (说话) → Hysteresis (防抖)
```

### 9.2 三层判断

**L1 遮挡判断：**
- |yaw| > 60° → `SELF_OCCLUDED`（极端侧脸）
- BiSeNet face parsing 检测嘴部区域 lip 像素占比（每 track 每 5 次调用跑一次，中间复用）
  - lip_ratio < 0.10 → `OCCLUDED`
  - lip_ratio 0.10~0.25 → PARTIAL（prob 压低 60%）

**L2 说话概率：**
- MediaPipe FaceLandmarker 提取 27 个嘴部 blendshape + yaw/pitch/roll
- 维护最近 12 帧的 ring buffer
- 计算滑窗特征（当前值/均值/标准差/极差/差分均值/最小/最大 = 210 维）
- XGBoost 模型输出 speaking_prob（0~1）
- 模型通过 52000 帧录制数据训练，F1 = 0.948

**L3 Hysteresis 状态机（数据驱动参数）：**
- prob ≥ 0.70 连续 3 帧 → `SPEAKING`
- prob < 0.30 连续 3 帧 → `NOT_SPEAKING`
- 45°+ 侧脸时自适应增加确认帧数（每超 5° 加 1 帧）

### 9.3 输出

| 状态 | 标签 | 含义 |
|------|------|------|
| speaking | SPK | 正在说话 |
| not_speaking | （无标签） | 没在说话 |
| occluded | OCC | 嘴部被遮挡 |
| self_occluded | S-OCC | 极端侧脸 |
| unobservable | ? | 无法检测人脸 |

### 9.4 性能影响

- 关闭时：零开销
- 开启时：在异步线程运行，不阻塞主循环。BiSeNet 每 5 帧跑一次（~60ms/次），XGBoost <0.1ms

## 10. 性能保护机制

### 10.1 三层 Embedding 限频

| 层级 | 机制 | 默认值 |
|------|------|--------|
| 每帧数量限制 | 单帧最多 N 次 embedding | 5 次/帧 |
| 每轨迹总次数上限 | 足够生成模板后停止 | 5 次/轨迹 |
| 时间冷却 | 同一轨迹两次嵌入最短间隔 | 200ms |

### 10.2 异步身份计算（IdentityWorker）

对齐、质量评估、采样、embedding、模板聚合、person 匹配、身份判定全部从主循环移到异步线程。主线程只读缓存标签。新人入画时 FPS 不波动。

### 10.3 CPU 核心分配

- SCRFD 检测器：`cpu_count - 2` 个核心
- ArcFace 嵌入器：2 个核心
- 两者不互相争抢，消除了冷启动 FPS 下降

### 10.4 Fail-Open

每一步都有独立异常捕获，任何出错只记日志不影响视频流。

## 11. Web 前端 + 主循环

### 11.1 架构

- Flask Web 框架，REST API + MJPEG 视频流
- 单页 HTML 前端，AJAX 交互

### 11.2 主循环（每帧）

```
1. 读帧
2. 判断是否为检测帧
3. 检测帧: SCRFD检测 + BoT-SORT跟踪
   非检测帧: 仅Kalman预测
4. 异步提交: IdentityWorker (身份) + MouthWorker (说话)
5. 读取异步结果缓存
6. 绘制跟踪框 + 标签 + HUD
7. JPEG编码 → MJPEG推流
```

### 11.3 标签含义

```
T#102 P#1 KS R#1 C3 SPK (0.87)
```

| 部分 | 含义 |
|------|------|
| `T#102` | 轨迹 ID |
| `P#1` | 人员 ID |
| `KS` | 身份状态（KS=已知, AMB=模糊, US=陌生人） |
| `R#1` | 已注册身份 ID |
| `C3` | 信用积分 |
| `SPK` | 说话状态（SPK=说话, OCC=遮挡, S-OCC=自遮挡） |
| `(0.87)` | 余弦相似度 |

### 11.4 前端开关

| 开关 | 控制内容 |
|------|----------|
| 启用人脸对齐采样 | Module 3 |
| 启用人员匹配 (T#→P#) | Module 4 |
| 启用信用积分制 | Credit Gate |
| 启用候选池 (C池) | Module 5 |
| 启用说话检测 (SPK/OCC) | 说话检测模块 |

```

│  摄像头 / 视频文件                                                    │

│     ▼                                                                │

│     │                                                                │

│     │         ┌────────────────────┤                                 │
│     │         │                    │                                 │
│     │         ▼                    ▼                                 │
│     │  [异步] IdentityWorker  [异步] MouthWorker (可选)              │

│     │    → Credit Gate            → BiSeNet 遮挡                     │

│     │    → 模板聚合               → Hysteresis                       │

│     │         │                    │                                 │
│     │         └────────┬───────────┘                                 │
│     │                  │                                             │
│     ▼                  ▼                                             │
│  [主线程] 读取缓存标签 → 绘制画面 → JPEG → 浏览器                     │
└─────────────────────────────────────────────────────────────────────┘
```
